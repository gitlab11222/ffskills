# Checklist生成 Skill

根据项目需求生成验收Checklist、发布Checklist、部署Checklist等。

## 使用方式

```
/create-checklist --type <类型>
/create-checklist --type验收 --project <项目名>
/create-checklist --type发布 --version <版本号>
/create-checklist --type部署 --env <环境>
```

## 参数说明

| 参数 | 格式 | 说明 | 示例 |
|------|------|------|------|
| --type | 验收/发布/部署/上线/回滚 | Checklist类型 | 验收 |
| --project | 项目名 | 项目名称 | 商品管理 |
| --version | 版本号 | 发布版本 | V1.0.0 |
| --env | 环境 | 部署环境 | 生产 |

## Checklist类型

| 类型 | 用途 | 适用阶段 |
|------|------|----------|
| 验收 | 产品验收检查 | 产品验收阶段 |
| 发布 | 发布前检查 | 发布上线阶段 |
| 部署 | 环境部署检查 | 部署阶段 |
| 上线 | 上线后验证 | 发布后验证 |
| 回滚 | 回滚操作检查 | 回滚阶段 |

## 执行步骤

### 1. 分析检查范围

#### 1.1 根据类型确定检查维度
- 验收Checklist: 功能、UI、性能、安全
- 发布Checklist: 代码、测试、环境、文档、通知
- 部署Checklist: 基础设施、应用配置、安全配置、监控
- 上线Checklist: 基础验证、功能验证、性能验证、安全验证
- 回滚Checklist: 数据恢复、服务恢复、功能验证

### 2. 生成检查项

#### 2.1 检查项分类
- 必查项（🔴）：必须检查，缺失不可继续
- 建议查项（🟡）：建议检查，缺失需记录
- 可选查项（🟢）：可选检查

### 3. 输出Checklist文档
