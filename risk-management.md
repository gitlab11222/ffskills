# 风险管理规范

## 风险识别

### 风险分类
| 类型 | 说明 | 示例 |
|------|------|------|
| 技术风险 | 技术实现难度、技术债务 | 新技术不熟悉、性能瓶颈 |
| 业务风险 | 业务复杂度、需求变更 | 业务规则复杂、需求频繁变更 |
| 资源风险 | 人员、时间、预算 | 人员变动、资源不足 |
| 时间风险 | 里程碑延期、依赖阻塞 | 关键路径延期、外部依赖延迟 |
| 外部风险 | 第三方服务、政策法规 | 供应商问题、合规要求 |

### 风险识别维度
| 维度 | 检查项 | 识别方法 |
|------|------|----------|
| 技术风险 | 新技术使用比例 > 30% | 技术评审 |
| 技术风险 | 技术债务 > 5项 | 代码审查 |
| 技术风险 | 性能瓶颈 > 3处 | 性能评估 |
| 业务风险 | 业务规则 > 20条 | 需求分析 |
| 业务风险 | 需求变更率 > 20% | 需求跟踪 |
| 资源风险 | 团队规模 < 需求 | WBS评估 |
| 资源风险 | 关键人员离职风险 | 团队评估 |
| 时间风险 | 关键路径 > 3个依赖 | 依赖分析 |
| 外部风险 | 第三方依赖 > 5个 | 依赖梳理 |

## 风险评估

### 风险等级定义
| 等级 | 概率 | 影响 | 风险评分 | 说明 |
|------|------|------|----------|------|
| 🔴 高风险 | 高(>60%) | 高 | ≥ 15 | 立即处理 |
| 🟡 中风险 | 中(30-60%) | 中/高 | 8-14 | 优先处理 |
| 🟢 低风险 | 低(<30%) | 低/中 | ≤ 7 | 监控观察 |

### 风险评分矩阵
```
          影响程度
          低    中    高
概率 高   🟡    🟡    🔴
概率 中   🟢    🟡    🔴
概率 低   🟢    🟢    🟡

评分: 低=1, 中=2, 高=3
风险评分 = 概率 × 影响
高风险 ≥ 9分, 中风险 4-6分, 低风险 ≤ 3分
```

## 风险应对

### 应对策略
| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 规避 | 消除风险源 | 高风险、可控 |
| 转移 | 转移给第三方 | 外部风险 |
| 减轻 | 降低概率或影响 | 中高风险 |
| 接受 | 接受并监控 | 低风险 |

## DO - 推荐做法

```markdown
// ✓ 项目启动时识别风险
# 风险识别清单
技术风险、业务风险、资源风险、时间风险、外部风险

// ✓ 风险有等级评估
风险评分 = 概率 × 影响
🔴 高风险 ≥ 9分

// ✓ 风险有应对方案
预防措施 + 应急方案

// ✓ 风险有责任人
每个风险有明确责任人

// ✓ 风险有监控指标
定期监控风险状态

// ✓ 风险有预警机制
超过阈值立即预警

// ✓ 风险有复盘总结
项目结束后风险复盘
```

## 最佳实践

### 风险识别全面化
- 多维度识别
- 定期更新风险清单
- 团队共同参与

### 风险应对具体化
- 预防措施可执行
- 应急方案可落地
- 责任人明确

### 风险监控常态化
- 定期监控指标
- 预警及时响应
- 状态持续更新
