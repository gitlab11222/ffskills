# 质量门禁规范

## 门禁定义

### 门禁概念
质量门禁是项目各阶段的准入/准出标准，确保每个阶段交付物质量达标。

### 门禁类型
| 类型 | 说明 | 适用阶段 |
|------|------|----------|
| 准入门禁 | 进入下一阶段前必须满足 | 需求评审、转测试、发布上线 |
| 准出门禁 | 离开当前阶段前必须满足 | 需求评审、测试验收、发布上线 |

## 门禁检查流程

### 检查步骤
```
准备材料 → 自检确认 → 门禁检查 → 问题整改 → 重新检查 → 通过确认
```

### 检查记录模板
```markdown
# 门禁检查记录

## 基本信息
- **项目名称**: {项目}
- **检查阶段**: {阶段}
- **检查时间**: {YYYY-MM-DD HH:mm}
- **检查人**: {姓名}

## 检查结果

### 检查清单
| 检查项 | 结果 | 备注 |
|--------|------|------|
| {检查项1} | ✅ 通过 | - |
| {检查项2} | ❌ 不通过 | {原因} |

### 问题清单
| 问题 | 级别 | 负责人 | 整改时间 |
|------|------|--------|----------|
| {问题} | 🔴/🟡/🟢 | {姓名} | {时间} |

## 检查结论
- **检查结果**: ✅ 通过 / ❌ 不通过 / ⚠️ 需整改后通过
- **下一步**: 进入下一阶段 / 整改后重新检查

## 签字确认
| 角色 | 姓名 | 签字 | 日期 |
|------|------|------|------|
| 检查人 | | | |
| 负责人 | | | |
```

## DO - 推荐做法

```markdown
// ✓ 每阶段有门禁检查
# 需求评审门禁
# 技术评审门禁
# 转测试门禁
# 测试验收门禁
# 产品验收门禁
# 发布上线门禁

// ✓ 门禁有检查清单
- [ ] PRD文档完整
- [ ] 原型设计完成

// ✓ 门禁有签字确认
| 角色 | 姓名 | 签字 | 日期 |

// ✓ 门禁不通过要整改
问题清单 + 整改时间

// ✓ 关键门禁多角色审批
发布上线：研发+测试+产品+项目经理
```

## DON'T - 禁止做法

```markdown
// ✗ 无门禁检查
// 直接进入下一阶段  // 禁止

// ✓ 正确做法
门禁检查 → 通过确认 → 进入下一阶段

// ✗ 门禁无检查清单
// 无具体检查项  // 禁止

// ✓ 正确做法
- [ ] PRD文档完整
- [ ] 原型设计完成

// ✗ 门禁无签字
// 无确认记录  // 禁止

// ✓ 正确做法
| 角色 | 姓名 | 签字 | 日期 |

// ✗ 门禁不通过继续
// 强制进入下一阶段  // 禁止

// ✓ 正确做法
整改 → 重新检查 → 通过确认

// ✗ 关键门禁单人审批
// 仅一人审批  // 禁止

// ✓ 正确做法
多角色审批：研发+测试+产品+项目经理
```

## 最佳实践

### 门禁前置化
- 提前准备检查材料
- 自检确认后再提交
- 预留整改时间

### 门禁标准化
- 统一检查清单模板
- 统一签字确认流程
- 统一问题整改流程

### 门禁可追溯
- 检查记录存档
- 问题整改跟踪
- 签字确认留痕
