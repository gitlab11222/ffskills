# 测试用例生成 Skill

根据需求文档或 API 定义自动生成测试用例，覆盖正向、逆向、边界、权限测试场景。

## 使用方式

```
/generate-testcase <需求ID>
/generate-testcase --from-prd <需求文档路径>
/generate-testcase --from-api <模块名>
/generate-testcase --from-fields <字段列表>
```

## 参数说明

| 参数 | 格式 | 说明 | 示例 |
|------|------|------|------|
| 需求ID | 数字 | TAPD 需求ID | 12345 |
| --from-prd | 文件路径 | PRD 文档路径 | docs/prd.md |
| --from-api | 模块名 | API 模块名 | user, order |
| --from-fields | 字段列表 | 字段定义 | name:string,age:int |

## 选项

- `--priority P0/P1/P2` - 指定生成的用例优先级范围
- `--type 功能/安全/性能` - 指定测试类型
- `--output 文件路径` - 指定输出文件路径

## 执行步骤

### 1. 解析输入源

#### 从 PRD 解析
- 提取功能模块
- 提取字段定义
- 提取业务规则
- 提取状态流转

#### 从 API 解析
- 解析接口 URL
- 解析请求参数
- 解析响应结构
- 解析业务逻辑

#### 从字段解析
- 解析字段类型
- 解析校验规则
- 解析默认值
- 解析枚举值

### 2. 生成用例场景

#### 正向用例（正常流程）
- 正常创建
- 正常查询
- 正常更新
- 正常删除
- 状态正常流转

#### 逆向用例（异常流程）
- 参数缺失
- 参数格式错误
- 业务规则违反
- 依赖条件不满足
- 状态异常流转

#### 边界用例（临界值）
- 数值边界（最小值、最大值）
- 字符长度边界
- 时间边界（开始时间、结束时间）
- 数量边界（0、最大值）

#### 权限用例
- 无权限访问
- 越权访问
- 角色权限边界

### 3. 编写测试用例

#### 用例结构
```markdown

- **前置条件**: {条件}

### 测试步骤
| 步骤 | 操作 | 预期结果 |
|------|------|----------|

|------|-----|------|

### 验证点
- [ ] {验证点}
```

## 生成规则

### 字段类型测试矩阵

| 字段类型 | 正向测试 | 逆向测试 | 边界测试 |
|----------|----------|----------|----------|
| string | 正常字符串 | 空值、特殊字符、超长 | 最小长度、最大长度 |
| int | 正常整数 | 空值、非数字、负数 | 最小值、最大值、0 |
| long | 正常长整数 | 空值、非数字 | 最小值、最大值 |
| decimal | 正常小数 | 空值、非数字、负数 | 最小值、最大值、精度边界 |
| bool | true/false | 空值、非布尔值 | - |
| datetime | 正常日期 | 空值、格式错误、无效日期 | 最小日期、最大日期 |
| enum | 有效枚举值 | 空值、无效枚举值 | - |

### 业务规则测试矩阵

| 规则类型 | 正向测试 | 逆向测试 |
|----------|----------|----------|
| 必填校验 | 填写必填字段 | 不填必填字段 |
| 格式校验 | 符合格式 | 不符合格式 |
| 范围校验 | 在范围内 | 超出范围 |
| 依赖校验 | 依赖满足 | 依赖不满足 |
| 状态校验 | 状态允许 | 状态不允许 |

### 状态流转测试矩阵

| 测试场景 | 测试方法 |
|----------|----------|
| 正常流转 | 按流程顺序执行状态变更 |
| 逆向流转 | 尝试非法状态变更 |
| 重复操作 | 尝试重复执行同一状态变更 |
| 并发操作 | 并发执行状态变更 |
