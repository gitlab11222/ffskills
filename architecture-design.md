# 架构设计规范

## 架构设计流程

### 设计阶段
```
需求分析 → 技术选型 → 架构设计 → 详细设计 → 设计评审 → 设计输出
```

| 阶段 | 输入 | 输出 | 负责人 |
|------|------|------|--------|
| 需求分析 | PRD | 技术需求清单 | 架构师 |
| 技术选型 | 技术需求 | 技术选型方案 | 架构师 |
| 架构设计 | 技术选型 | 架构设计文档 | 架构师 |
| 详细设计 | 架构设计 | 详细设计方案 | 研发 |
| 设计评审 | 详细设计 | 评审确认 | 全员 |
| 设计输出 | 评审确认 | 设计文档归档 | 架构师 |

```markdown
## 3. 系统架构

### 3.1 架构风格
- **架构类型**: 微服务 / 单体 / 混合
- **部署模式**: 容器化 / 虚拟机 / 云原生

### 3.2 系统架构图
```
┌─────────────────────────────────────────────────────┐
│                    用户层                            │
│    Web App    Mobile App    Admin Portal           │
└─────────────────────────────────────────────────────┘
                        ↓ HTTPS
┌─────────────────────────────────────────────────────┐
│                    网关层                            │
│         Nginx / Kong / API Gateway                  │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                    服务层                            │
│  User Service  Order Service  Product Service      │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   数据层                             │
│      MySQL       Redis        Message Queue        │
└─────────────────────────────────────────────────────┘
```

### 3.3 服务拆分策略
| 服务 | 职责 | 依赖 | 数据库 |
|------|------|------|--------|
| User Service | 用户管理 | 无 | user_db |
| Order Service | 订单管理 | User/Product | order_db |
| Product Service | 商品管理 | 无 | product_db |

### 3.4 服务通信方式
| 场景 | 方式 | 说明 |
|------|------|------|
| 同步调用 | REST API | 请求响应 |
| 异步调用 | 消息队列 | 解耦异步 |

## 7. 安全设计

### 7.1 安全架构
```
┌────────────┐
│  安全边界  │  WAF / Firewall

```
### 7.2 安全措施
```
