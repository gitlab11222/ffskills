# 变更管理规范

## 变更类型

### 变更分类
| 类型 | 说明 | 影响范围 | 审批级别 |
|------|------|----------|----------|
| 需求变更 | 功能增删改 | 功能范围 | 产品负责人 |
| 技术变更 | 技术方案调整 | 技术架构 | 架构师/技术经理 |
| 排期变更 | 时间节点调整 | 项目进度 | 项目经理 |
| 资源变更 | 人员/预算调整 | 资源配置 | 项目经理 |
| 环境变更 | 环境配置调整 | 运行环境 | 运维 |

### 变更等级
| 等级 | 定义 | 影响评估 | 审批流程 |
|------|------|----------|----------|
| 🔴 重大变更 | 核心功能变更、架构调整 | 影响范围 > 30% | 多角色审批 |
| 🟡 中等变更 | 重要功能变更、排期调整 | 影响范围 10-30% | 负责人审批 |
| 🟢 轻微变更 | 非核心功能、文案调整 | 影响范围 < 10% | 快速审批 |

## 变更流程

### 标准流程
```
变更申请 → 变更评估 → 变更审批 → 变更执行 → 变更验证 → 变更关闭
```

### 流程说明
| 步骤 | 说明 | 负责人 | 输出 |
|------|------|--------|------|
| 变更申请 | 提交变更请求 | 申请人 | 变更申请单 |
| 变更评估 | 评估影响范围 | 评估人 | 评估报告 |
| 变更审批 | 审批变更请求 | 审批人 | 审批确认 |
| 变更执行 | 执行变更内容 | 执行人 | 变更记录 |
| 变更验证 | 验证变更效果 | 验证人 | 验证结果 |
| 变更关闭 | 关闭变更记录 | 管理人 | 变更关闭 |

## 快速变更流程

### 适用场景
- 文案修改
- 配置调整
- Bug修复
- 非核心优化

### 快速流程
```
申请 → 评估(快速) → 审批(单人) → 执行 → 验证 → 关闭
```

### 快速审批条件
| 条件 | 说明 |
|------|------|
| 影响范围 | < 10% |
| 工时消耗 | < 1人天 |
| 进度延迟 | < 1天 |
| 无风险 | 无技术/业务风险 |

## 紧急变更流程

### 适用场景
- 生产环境故障
- 安全漏洞修复
- 紧急业务需求

### 紧急流程
```
申请 → 审批(快速) → 执行 → 验证 → 补充记录
```

### 紧急审批条件
- 生产环境故障需立即修复
- 审批后补正式流程
- 执行记录补完整

## 最佳实践

### 变更流程规范化
- 标准流程：申请→评估→审批→执行→验证→关闭
- 快速流程：申请→评估→审批→执行→验证→关闭
- 紧急流程：申请→审批→执行→验证→补充记录

### 变更审批分级化
- 重大变更：多角色审批
- 中等变更：负责人审批
- 轻微变更：快速审批

### 变更记录追溯化
- 变更申请单存档
- 变更执行记录存档
- 变更登记表维护
