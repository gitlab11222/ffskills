# 验收规范

## 验收流程

### 验收阶段
```
研发自测 → 转测试 → 测试验收 → 产品验收 → 发布验收 → 上线验收
```

| 阶段 | 负责人 | 输入 | 输出 | 检查要点 |
|------|--------|------|------|----------|
| 研发自测 | 研发 | 功能代码 | 自测报告 | 功能可用性 |
| 转测试 | 研发 | 自测通过 | 转测试单 | 自测完成 |
| 测试验收 | 测试 | 转测试单 | 测试报告 | 功能、安全、性能 |
| 产品验收 | 产品 | 测试报告 | 验收确认 | 功能、UI、流程 |
| 发布验收 | 运维 | 验收确认 | 发布报告 | 环境、部署 |
| 上线验收 | 运维/运营 | 发布完成 | 上线确认 | 功能可用、数据正常 |

## 验收问题处理规范

### 问题分级

| 级别 | 定义 | 处理方式 |
|------|------|----------|
| 🔴 阻塞 | 核心功能不可用 | 立即修复，重新验收 |
| 🟡 严重 | 重要功能异常 | 发布前修复 |
| 🟢 一般 | 非核心问题 | 下版本修复 |

## 最佳实践

### 验收清单化
- 每次验收使用标准清单
- 每项检查明确结果

### 验收签字化
- 关键验收需签字确认
- 多角色联合验收

### 问题追溯化
- 问题编号管理
- 处理状态跟踪

### 遗留规范化
- 遗留问题有规避方案
- 后续计划明确
